WiFi Captive Portal with Federated Filesystem Security
=======================================================
Implementation Instructions
============================

Project:  WP1 (Web Programming) + SEC-PRJ-2_23 (System Security)
Platform: Raspberry Pi OS Bookworm (64-bit) + Docker
Author:   Mohammadamin Jahanimajd

──────────────────────────────────────────────────────────────────────────────
PART A — Local / Docker Demo (any Linux/macOS/Windows machine)
──────────────────────────────────────────────────────────────────────────────

Requirements
  • Docker Desktop (or Docker Engine + Docker Compose v2)
  • Port 5000 and 5001 free on your machine

Step 1 — Build and start the two-node cluster
  cd Captive_Portal
  docker compose up --build

  This starts two containers:
    captive_portal  → http://localhost:5000   (primary portal node)
    neighbor_node   → http://localhost:5001   (federation neighbour)

  On startup both nodes:
    a. Generate an RSA-2048 key pair (persisted in Docker volume).
    b. Generate a 32-byte AES-256 master key and split it into shards.
    c. Exchange shards with each other via /federation/provide-shard.
    d. Reconstruct the master key and "mount" the secure partition
       (writes an AES-GCM encrypted marker file to the secure_store volume).

Step 2 — Open the captive portal
  http://localhost:5000

  You will see a login/register page.

Step 3 — Register a new user
  • Click the "Register" tab.
  • Fill in a username and password (≥ 6 characters).
  • Click "Create Account & Connect".
  • You are redirected to your personal dashboard showing device info
    and the current federation/security status.

Step 4 — Login as admin
  • Go to http://localhost:5000/logout to clear your session.
  • Go to http://localhost:5000 and login with:
      Username : admin
      Password : admin123
  • You are redirected to the Admin Panel.

Step 5 — Explore the Admin Panel
  Users tab:
    • View all registered users.
    • Activate / deactivate accounts with the pause/play button.
    • Delete users with the trash button.

  Devices tab:
    • All devices that have visited the portal are listed.
    • Authorise or revoke access per device.
    • On a real Pi, authorising calls setup_iptables.sh to add an iptables rule.

  Federation Nodes tab:
    • Add neighbour nodes by clicking "Add Node" and filling in the form.
    • Trust/untrust nodes — only trusted nodes can exchange key shards.
    • The neighbour container (node-2) can be added manually here.

  System tab:
    • Shows partition mount status, shards collected, master key status.

Step 6 — Check the federation API
  curl http://localhost:5000/api/status
  curl http://localhost:5000/federation/info
  curl http://localhost:5001/federation/info

Step 7 — Stop the cluster
  docker compose down

──────────────────────────────────────────────────────────────────────────────
PART B — Raspberry Pi Deployment
──────────────────────────────────────────────────────────────────────────────

Hardware required
  • Raspberry Pi 3B+ / 4B / 5
  • SD card ≥ 16 GB
  • Ethernet cable (for internet uplink on eth0)
  • WiFi built-in or USB WiFi adapter (for the hotspot on wlan0)

Step 1 — Flash OS
  Download Raspberry Pi OS Bookworm (64-bit Lite) from raspberrypi.com.
  Flash to SD card with Raspberry Pi Imager.
  Enable SSH in the imager settings.

Step 2 — First boot & updates
  ssh pi@<pi-ip>
  sudo apt update && sudo apt upgrade -y
  sudo apt install -y docker.io docker-compose-plugin git

Step 3 — Copy project to the Pi
  From your machine:
    scp -r Captive_Portal pi@<pi-ip>:~/Captive_Portal
  Or clone the repo directly on the Pi.

Step 4 — Set up the WiFi hotspot
  sudo bash ~/Captive_Portal/scripts/setup_hotspot.sh

  This installs hostapd + dnsmasq and configures:
    • SSID: CaptivePortal-IoT
    • Password: SecurePortal2024  ← CHANGE THIS in the script first!
    • IP range: 192.168.73.10 – 192.168.73.200
    • Portal IP: 192.168.73.1

Step 5 — Set up iptables redirect rules
  sudo bash ~/Captive_Portal/scripts/setup_iptables.sh

  Any device connecting to the hotspot will now be redirected to
  http://192.168.73.1:5000 for login/registration.

Step 6 — (Optional) Initialise the LUKS secure partition
  This step is only needed on first boot or if you want a real encrypted
  partition.  Skip if running in demo/Docker mode only.

  a. Identify the partition you want to encrypt:
       lsblk
     (e.g. /dev/mmcblk0p3 — create this partition first with fdisk if needed)

  b. Edit mount_secure_fs.sh and set DEVICE to your partition.

  c. Run the create command (DESTROYS ALL DATA on that partition):
       sudo bash ~/Captive_Portal/scripts/mount_secure_fs.sh create

  d. The script will create the LUKS container, format it ext4, and mount it.

Step 7 — Launch the portal
  cd ~/Captive_Portal

  Edit docker-compose.yml to change ADMIN_PASSWORD and SECRET_KEY before
  going live!

  docker compose up -d

  The portal is now live at http://192.168.73.1:5000.

Step 8 — Verify operation
  a. Connect your phone/laptop to the "CaptivePortal-IoT" WiFi network.
  b. Open any HTTP website — you should be redirected to the portal.
  c. Register or login — after success you have internet access.
  d. On the Pi, check logs:
       docker logs captive_portal

Step 9 — Adding a second Pi as a federation neighbour
  a. Repeat steps 1–7 on a second Raspberry Pi.
  b. In each Pi's docker-compose.yml, set NEIGHBOR_NODES to the other Pi's IP:
       Pi 1: NEIGHBOR_NODES=<pi2-ip>:5000
       Pi 2: NEIGHBOR_NODES=<pi1-ip>:5000
  c. Restart both nodes:
       docker compose down && docker compose up -d
  d. In each portal's Admin Panel → Federation Nodes tab, add the other node
     and set it as "Trusted".
  e. Restart so bootstrap runs again — both nodes will collect each other's
     shards and mount their secure partitions.

PART C — Development Mode (run Flask directly without Docker)
──────────────────────────────────────────────────────────────────────────────

Requirements
  • Python 3.11+

Step 1 — Install dependencies
  cd Captive_Portal/backend
  python -m venv .venv
  .venv\Scripts\activate   # Windows
  source .venv/bin/activate  # Linux/macOS
  pip install -r requirements.txt

Step 2 — Set environment variables
  export FLASK_ENV=development
  export SECRET_KEY=dev-secret-do-not-use-in-production
  export DATABASE=/tmp/portal.db
  export DATA_DIR=/tmp/portal_data
  export NODE_ID=node-dev
  export ADMIN_USERNAME=admin
  export ADMIN_PASSWORD=admin123

Step 3 — Run
  python app.py

  Open http://localhost:5000

Troubleshooting
──────────────────────────────────────────────────────────────────────────────

Q: The captive portal does not redirect my device.
A: Check that hostapd and dnsmasq are running:
     systemctl status hostapd dnsmasq
   Check iptables rules:
     sudo iptables -t nat -L PREROUTING -v

Q: The Docker build fails.
A: Make sure you are in the Captive_Portal directory (not backend/) when
   running docker compose up.

Q: The secure partition shows "Locked" even after both nodes started.
A: The federation nodes need to be added to each other's trusted list in the
   Admin Panel (Federation Nodes tab → Add Node → toggle Trust).
   Then restart the containers so bootstrap re-runs.

Q: I forgot the admin password.
A: Delete the database volume and restart:
     docker compose down -v
     docker compose up -d
   This resets all data and creates a fresh admin account.
